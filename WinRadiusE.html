Setup guide:

  • Launch WinRadius
  • Click "Configure ODBC automatically" button at "Settings/Database...".
  • Restart WinRadius. Now, all settings of WinRadius is OK.
  • Add some users to WinRadius by clicking "+" toolbar button.
  • Configure your NAS to WinRadius. Please go here for the details.
  • That's all. If these users access to NAS, authentication and accounting information will logged to WinRadius and be saved to database. Now, you can use RadiusTest to test your WinRadius , please recheck the Radius IP, port and user name. (RadiusTest's secret is default secret, and the user's password is: "password").
  • Good luck with you.

Performance tune:

  • Use better hardware, especially use larger memory.
  • Tune your database to fasten database. More information please refer to your database's guide.
  • Show less Log information.

Skills:

  • Can restore the origin configuration by delete WinRadius.config. If you changed configuration, please restart WinRadius to make it active.
  • All backup information of database is saved in  WinRadius.backup file.
  • Null password of a user means all password is acceptable.
  • WinRadius server's interface can minimized as an icon of task bar; you can also restore it by click the icon.
  • The communication between WinRadius client and server is ciphered with system NAS password, please make sure the WinRadius client and server have used the same system NAS secret.
  • You can use "Advanced/Create Radius Table" to create the table structure for your database.
  • Make sure your WinRadius server and client is the same version.
  • If your database shutdown while your WinRadius is running, never mind, WinRadius server will automatic backup all information. What you need do is only do "recover" operation when your database is OK.
